Pythonによる簡単なあいまい歌詞検索（「チャントオシエール」製作 vol.2）

今回は、以前作成したヴィッセル神戸チャント検索ボット「チャントオシエール」のチャント検索部分について、どのような仕組みで実現しているかを書こうと思います。システム全体の概要図は「ヴィッセル神戸チャント検索ボット「チャントオシエール」製作 vol.1」で触れていますので、合わせてどうぞ。

「あいまい」検索をしたい！

私が作成したチャント検索ボット「チャントオシエール」では、歌詞や選手名でのあいまい検索を（一応）可能にしています。例えば、「イニエスタ」のことを「イエニスタ」と言ったとしても検索できます。これは、よくある言い間違いをあらかじめ用意しているのではなく、多少の工夫をして解決しています。

このあいまい検索を行うための基本的な処理の流れは以下の通りになります。

  1. 検索対象となる選手名やチャントの歌詞とクエリのカタカナ化
  2. N-gramを基本とした対象データのランキング

なぜカタカナ化するかというと、歌詞検索をする際に、「漢字じゃないと検索できません」となってしまうと、検索ハードルが上がるからです。聞こえてきたチャントを検索したり、知ってる一部分のチャントだけで検索するために、日本語で入力された場合はカタカナに置き換えるようにしました。

カタカナ化に用いるあれこれ

検索を行う際にクエリ（検索キーワード）をカタカナにし、 あらかじめカタカナ化している歌詞と比較することで検索を行っています（検索部分については後ほど述べます）。 歌詞や選手名をあらかじめカタカナ化をするのは人手ですれば解決しますが、 ユーザがLINEで投げるクエリは、システム側で処理を行いカタカナにする必要があります。今回主に利用したのは、Pythonパッケージの「Janome」と「Jaconv」です。 これらはpipコマンドでインストール可能です。

pip install janome
pip install jaconv

Janomeについて

JanomeはPythonで簡単に使える日本語の形態素解析器です。公式のページにある簡単な使い方は以下のとおりです。形態素解析を使うことで、日本語を形態素と呼ばれる最小単位に分解し、形態素ごとの様々な情報を見ることができます。この中の情報で、「読み」を表す各行の後ろから2つ目の要素を利用しカタカナ化します。 

from janome.tokenizer import Tokenizer
t = Tokenizer()
for token in t.tokenize(u'すもももももももものうち'):
    print(token)

このJanomeを利用して、以下のような関数を作成しました。 この関数を使えば、形態素解析結果をリストとして受け取ることができます。 

from janome.tokenizer import Tokenizer
janome = Tokenizer()
def janome_lst_out(text):
    """
    形態素解析してリスト型として出力
    """
    morph_lst = []
    for line in janome.tokenize(text):
        morph_dict = {}
        temp_lst = str(line).split("\t")
        if len(temp_lst) == 2:

            add_lst = temp_lst[1].split(",")
            morph_dict["surface"] = temp_lst[0]
            morph_dict["base"] = add_lst[-3]
            morph_dict["pos"] = add_lst[0]
            morph_dict["pos1"] = add_lst[1]
            morph_dict["kata"] = add_lst[-2]
            morph_lst.append(morph_dict)
    return morph_lst

用意した辞書を用いてカタカナ化

Janomeだけではなく、あらかじめこちらが用意した「漢字→カタカナ」の変換辞書を用いてカタカナ化します。Janomeでは公式サイトで以下に書かれている辞書を利用して形態素解析を行っています。このため、選手名や歌詞の一部は、一般的な漢字の読み方と違うものがあります。これらをあらかじめこちらで準備しておきます。

内包辞書として mecab-ipadic-2.7.0-20070801 を使っています

テキストフォルダに以下のようにカンマ区切りで漢字とカタカナのペアを保存してきます。例のように、選手の名前などはあらかじめ準備しておかないと標準の辞書では対応できません。なお、Janomeで利用する辞書の中に直接取り込むこともできるのですが、今回はしていません。

前川,マエカワ
黛也,ダイヤ
飯倉,イイクラ
大樹,ダイキ
吉丸,ヨシマル
絢梓,ケンシン
︙

これらのテキストファイルを、以下の関数で辞書型henkan_dictに格納していきます。漢字をキーにして、値にカタカナを入れます。この辞書型を用いて、与えられたテキストに対して、Janomeによる変換を行う前にカタカナ化してしまいます。 

def road_henkan_dict():
    henkan_fpath = "ファイル名"
    henkan_dict = {}
    with open(henkan_fpath, "r", encoding="utf8")as fr:
        for line in fr.readlines():
            key, val = line.rstrip("\n").split(",")
            henkan_dict[key] = val
    return henkan_dict

Jaconvによる前処理

入力されるクエリは全角や半角が混じっていたりするので、前処理をしないといけません。これを行うために、Jaconvを利用しています。Jaconvについて公式では、以下のように書かれています。

jaconv (Japanese Converter) はひらがな・カタカナ・全角・半角の文字種変換を高速に行います。 Pythonのみで実装されているので、Cコンパイラが使えない環境でも利用できます。

組み合わせたカタカナ化関数

  • janome
  • 辞書
  • jaconv

以上の3つを利用して、あるテキストが入力された時にカタカナを出力する関数を以下のように作成しました。この関数により、文字列を入力として受け取りカタカナ化して出力します。 

def kata_out(text):
    """
    入力されたtextをカタカナに変換して出力
    """
    kata_lst = []
    # あらかじめ用意した辞書による変換
    for word, kata in henkan_dict.items():
        text = text.replace(word, kata)
    # Jaconvによる前処理
    text = jaconv.z2h(jaconv.h2z(text), kana=False,
                        digit=True, ascii=True).upper()
    # Janomeで変換
    for morph in janome_lst_out(text):
        if morph["pos"] != "記号" and morph["pos1"] != "数":
            if morph["kata"] == "*":
                kata_lst.append(morph["surface"])
            else:
                kata_lst.append(morph["kata"])

    kata_text = jaconv.hira2kata("".join(kata_lst)).upper()

    return kata_text

スコア付け方法

以上のカタカナ化関数を用いて、LINEで送信されるたびにクエリをカタカナ化します。そして、あらかじめカタカナ化した歌詞や選手名と比較し、チャント毎にスコアを与えランキングします。本ページではスコア付けを中心に書きます。 単純な処理の流れは以下のとおりです。

  • カタカナ化したクエリのN-gram化
  • N-gram化したクエリによるスコア付け

N-gramについて

コトバンクでは以下のようになっています。 詳しくはこちらのQiita記事を読むと良いと思います。

任意の文字列や文書を連続したn個の文字で分割するテキスト分割方法．特に，nが1の場合をユニグラム（uni-gram），2の場合をバイグラム（bi-gram），3の場合をトライグラム（tri-gram）と呼ぶ．

文字列に対して、最大がその文字列の長さ、最小が1文字になる各N-gramを作成する関数を作成しました。上記で作成したカタカナ化する関数も利用して、入力された文字列を「カタカナ化してN-gramにする」か、「そのままN-gramにする」かの2種類で出力できます。例として「ヴィッセル神戸」を入力し、カタカナ化してN-gramで出力してみます。

def get_kata_ngram(string, mode=0):
    """
    入力された文字列を最大数としたn-gramの出力
    @text: ngramを取得する文字列
    """
    # カタカナ化する場合
    if mode == 0:
        kata_string = kata_out(string)
    else:
        kata_string = string
    srt_len = len(kata_string) + 1
    result = []
    for n in range(1, srt_len):
        result.append([kata_string[k:k+n]
                        for k in range(len(kata_string)-n+1)])
    return result[::-1]
# 実行してみる
get_kata_ngram("ヴィッセル神戸")

このようにして、入力された文字列の長さの7-gram（Septi-gram?）から、1文字のUni-gramまでの全組み合わせを配列に保存します。そしてこのN-gramリストを用いて検索（スコア付け）を行います。

スコア算出

上記のN-gram出力関数を用いて得られたN-gramリストを用いて、各チャントに対してスコアを与えます。先にスコアの算出を行うプログラムを載せておきます。 

 def scoring_by_ngram(doc, kata_ngrams_lst):
        """
        各n-gramで検索を行い文字数が多いものでマッチすれば高いスコアを返す
        @doc: スコア付け対象ドキュメント
        @ngram_lst: 検索するngram
        """
        max_len = len(kata_ngrams_lst)
        score = 0
        query = kata_ngrams_lst[0][0]
        match_score = len(set(query) & set(doc)) / len(set(query))

        for ngrams in kata_ngrams_lst:
            for ngram in ngrams:
                if ngram in doc:
                    score = len(ngram)/max_len * match_score
                    return score
        return score

このスコア付け関数は下のイメージ図のような2つのスコア（match_score=スコア2、len(ngram)/max_len=スコア1）を計算し、それらをかけ合わせたもの（最終スコア = スコア1 × スコア2）を最終的なスコアとして返しています。そして、1つのクエリに対して、検索対象全てにスコア付けを行い、そのスコアを基にランキングを作成し上位の結果の一部を表示させます。

実際に「チャントオシエール」でどのようなスコア付けされるかをいかに示します。クエリを「イエニスタ」にして、歌詞検索したときの結果になります。

順位 スコア チャント名 歌詞
1 0.40 イニエスタのチャント アンドレスイニエスタ ララララララ…
2 0.20 神戸讃歌 俺達のこの街に お前が生まれたあの日 …
3 0.16 Sha la la この歌声が聞こえるか 熱いクルヴァはここに…

このスコアを基に、スコア低い時に「ちょっと自信ないです」という文言をつけたりもしています。また、上位何件を表示するかに関しては以下の基本方針で実装しています（今回は具体的には触れません）。

  • スコアが1.0のものはすべて表示
  • n位のスコアとn+1位のスコアの差が閾値以上であればn+1位以降は表示しない

まとめ

今回は、ヴィッセル神戸チャント検索ボット「チャントオシエール」がどのように歌詞検索を行っているかについて簡単にまとめました。今後、チャントオシエールについて書くかはわかりませんが、色々改善点はあるのでそのあたりを書きたいとは思っています。例えば、「一度検索されたクエリは結果をMySQLに保存しておくことで、2回目以降同じクエリが入力された時に高速に検索できるようにする」などです。
