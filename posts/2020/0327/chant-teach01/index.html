ヴィッセル神戸チャント検索ボット「チャントオシエール」製作 vol.1

LINE Botを使ってヴィッセル神戸のチャント検索ができる 「チャントオシエール」を作成していこうと思います。 （実はすでに結構作っています） 作成に伴って記録的なものを始めようと思います。 今回は第1回なので、以下の内容を書こうと思います。

  • 作ろうと思ったきっかけ
  • 作成に用いるもの

作ろうと思ったきっかけ

修士論文の提出を終え、 修士生活最後の学会発表も終えて、 自由な時間ができたのに、 コロナの影響であまり家から出られないので、 なんか作ろっかなと思ったのがきっかけです。 「どうせなら自分の好きなことに関連させたいなぁ。」ということで、 ヴィッセル神戸に関するものを作ることにし、 せっかくなので、私がヴィッセル神戸にハマりだした時に、 スタジアムでの観戦時に感じていた不満を解決しようと思いました。 その不満が以下の2つです。

  • 「チャント歌いたいけど歌詞わからねぇ！」
  • 「歌詞検索したいけどWeb検索じゃ通信遅え！」

聞こえてくる熱いチャントを一緒に歌おうにも、 何を言っているのか、一部しかわからないことって多くないですか？ かといって、スマホで検索しようにも、通信が混雑してなかなかWeb検索できません。 ここで、

歌詞の一部からチャントを検索できるチャットボットがあれば便利なのでは？

と考えました。 聞こえてきたチャントの聞こえた部分だけで検索できれば、 一緒に歌うことが可能で、 チャットボットであればテキストベースの通信のため、 通信速度的問題も解決できる（はず）。 チャットボットで一番身近なものは LINEボット だと思うので （実装したことがあるのがLINEボット）、 LINEで「チャント検索」と「選手検索」ができるチャットボット 「チャントオシエール」を作成することにしました。

「チャントオシエール」について

右の画像が「チャントオシエール」君のプロフィール画像です。 知り合いに書いてもらった可愛いイラストを利用しています！ 実装する主な機能は以下の3つです。

  • 歌詞によるチャント検索
  • 名前や背番号、ポジションによる選手検索
  • 機能切り替えメニュー

まず、メイン機能である歌詞による チャント検索 は、 チャントの一部を「チャントオシエール」に送信することで、 それを含むチャントを送り返してくれる機能です。 次に、選手検索 は、名前や背番号、ポジションを送信することで、 選手の簡単な情報を表示する機能です。

また、これらの機能を切り替えるメニューを作成し、 利用ユーザ毎にどちらの機能を利用しているかをデータベースに保存することで、 ユーザ毎に検索機能を切り替えます。 開発段階ですが、イメージ図を貼っておきます。

作成に用いるもの

「チャントオシエール」君の作成に利用するものは大きく分けて以下の4つです。

  • LINE Messaging API（公式サイト）
  • Python3（line-bot-sdk-python）
  • MySQL
  • ngrok

かなり雑な処理のイメージ図はこんな感じです。


LINE Messaging API

主な役割

  • メッセージの送受信

LINEでチャットボットを作成するためのAPIです。 このAPIの応答メッセージ機能を使って、 利用者から検索したい内容が送られた時に、 検索結果を返信しています。


Python3

主な役割

  • APIを叩く
  • チャント、選手の検索機能
  • MySQLとの接続

今回は私のメイン使用言語であるPythonを利用してLINE Messaging APIを叩いていきます。 APIを叩くにあたってGithubで公開されている公式SDKを利用しています。 他のいくつかの言語にも対応しているようです。
検索機能についての詳細は今後の記事で触れる予定です。 この検索部分が今回のチャットボットのポイントになると思います。 例えば、イニエスタのチャントを検索したい時に、 間違って「イエニスタ」と入力してしまう人もいると思います （そんなにいっぱいの人に使ってもらえるかは置いておいて）。 このような場合でも、ちゃんとイニエスタのチャントを検索できるようにします。 また、後述のMySQLとのデータのやり取りもPythonを用いています。


MySQL

主な役割

  • ユーザの検索モードの保管

現在（2020年3月27日時点）では「チャント検索」と「選手検索」の2つのモードを実装予定で、 利用するユーザ毎にどちらのモードであるかの情報が必要になります。 今回はMySQLを使ってこれらの情報を管理することにしました。 DBをこういったサービス運用に利用したことがなかったので、 練習という意味も込めてMySQLを利用しています。


ngrok

主な役割

  • ローカルサーバで起動しているサービスの外部公開

昔使っていたノートPCにUbuntuを入れて簡易サーバ化し、 そこでSQLやらPythonやらを動かしています。 これらをngrokを使って外部に公開することで、 チャットボットを動かしています。 無料でアカウント作成をすることで時間制限なく利用できるため、 非常に便利なサービスです。

今後の予定

開発自体は結構進んでいるのですが、まとめる気力が起きないので、 頑張ってまとめて更新していこうと思います。
